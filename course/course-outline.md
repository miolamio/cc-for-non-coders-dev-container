### Занятие 4. Агенты, субагенты и slash-команды
Четверг, 5 марта, 19:00–20:30

До сих пор мы работали с Claude один на один. Но Claude Code умеет запускать копии себя — субагентов. У каждого свой контекст, свои инструменты, своя задача. Субагент отработал — вернул результат в основной сеанс. Это как отправить помощника за справкой: он уходит, приносит ответ, но с другими помощниками не разговаривает. А через slash-команды весь сценарий можно упаковать в одно слово: `/weekly-report` — и Claude сам знает, какие файлы взять, по какому шаблону собрать, куда положить.

**Что попробуем на практике:**
— Презентация из Excel: один субагент анализирует данные, другой собирает слайды. На выходе — 12 слайдов с графиками и выводами.
— Протокол встречи: скармливаем расшифровку часового созвона. Claude вытаскивает решения, задачи, дедлайны, ответственных. Итог — протокол и готовое сообщение для Telegram-чата.
— 15 персональных отчётов по 360-оценке: субагенты разбирают данные по каждому сотруднику одновременно.
— Лендинг по описанию: Claude верстает HTML-страницу с формой заявки и адаптивной вёрсткой.
— SEO-аудит блога: субагенты разбирают 80 статей параллельно — проверяют актуальность, структуру, ключевые слова. На выходе — список: что обновить, что убрать, о чём написать.
— Создаём свои slash-команды и hooks для повторяющихся задач.

Домашнее задание: решить задачу из своей работы с субагентами. Сделать slash-команду.

### Занятие 5. Agent Teams: агенты, которые общаются между собой
Понедельник, 9 марта, 19:00–20:30

Субагенты из прошлого занятия работают поодиночке: каждый отчитывается только «наверх», основному Claude. Друг о друге они ничего не знают. Agent Teams устроены иначе. Это несколько полноценных экземпляров Claude Code — каждый со своим контекстом, MCP-серверами, навыками. У них общий список задач с зависимостями и система сообщений: один нашёл проблему — написал другому напрямую, без посредников. Они спорят, проверяют работу друг друга, сами распределяют задачи. Больше похоже на настоящую команду, чем на набор помощников.

**Что попробуем на практике:**
— Telegram-бот для записи клиентов: один агент пишет логику, другой подключает Google Sheets, третий — документацию. Если у первого меняется структура данных — он сообщает об этом второму напрямую.
— Анализ 300 ответов из NPS-опроса: один агент разбирает по темам, другой оценивает тональность, третий собирает итоговый отчёт с цитатами. Агент тональности находит аномалию — сразу пишет аналитику, не дожидаясь конца.
— Cold outreach: из таблицы с 20 компаниями агенты параллельно готовят письма. Если один находит удачный приём — делится с остальными.
— Портфолио-сайт: один агент пишет тексты, другой верстает, третий добавляет форму. Верстальщик просит у текстовика уточнение — и получает его сам, без вашего участия.

## FAQ

### Будет ли запись?
Да. Записи доступны 3 месяца.

### Какой уровень подготовки нужен?
Опыт программирования не нужен. Если умеете описать задачу словами — справитесь.

### Нужно ли знание языков программирования?
Нет. Вы говорите Claude, что делать, на обычном русском. Он пишет код сам.

### Что нужно для обучения?
Компьютер с интернетом (Mac, Windows или Linux). Доступ к AI-модели GLM 4.7 включён в стоимость. Если есть своя подписка Claude — тоже подойдёт.

### Что входит в стоимость?
10 часов занятий, записи на 3 месяца, доступ к AI-модели, каталог из 60+ кейсов, Telegram-чат с поддержкой.

### У меня уже есть подписка Claude — что тогда?
Используйте её. А доступ к GLM пригодится для знакомства с альтернативными моделями.

### Что за альтернативные провайдеры?
Claude Code — это оболочка, которая работает с разными AI-моделями. На курсе основной провайдер — GLM 4.7: хорошо понимает русский, не требует зарубежной оплаты. Если есть подписка Claude — покажем, как подключить и её.

### Безопасно ли использовать Claude Code с рабочими данными?
Разбираем на первом занятии: что видит Claude Code, как ограничить доступ к папкам, какой провайдер выбрать, если данные нельзя отправлять за рубеж.
