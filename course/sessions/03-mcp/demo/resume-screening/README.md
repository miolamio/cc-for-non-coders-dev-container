# Демо: скрининг 50 резюме

## Что показываем

Claude получает файл с 50 кандидатами и описание вакансии, оценивает по критериям и выдает shortlist из 8 человек с обоснованием. Обычно такой скрининг занимает у рекрутера несколько часов, здесь -- пара минут.

Время на демо: скрининг ~1-2 минуты, с обсуждением и уточняющими вопросами ~10-12 минут.

## Файлы

- `resumes.csv` -- 50 кандидатов (имя, позиция, опыт, навыки, образование, город, зарплатные ожидания, краткое описание)
- `job-requirements.md` -- описание вакансии "Руководитель digital-маркетинга" с критериями оценки

## Основной промпт

```
У меня файл resumes.csv с 50 кандидатами на позицию "Руководитель digital-маркетинга". Требования к вакансии в файле job-requirements.md.

Проанализируй каждого кандидата по критериям из файла. Выдели shortlist из 8 лучших кандидатов. По каждому из них:
- Общая оценка (от 1 до 10)
- Почему подходит (конкретные пункты из опыта)
- Риски или слабые стороны
- Рекомендация: пригласить на собеседование / запасной вариант / отказать

Отдельно перечисли 5 кандидатов, которых точно не стоит приглашать, и почему.
```

## На что обратить внимание студентов

1. **Claude работает с CSV напрямую.** Не нужно конвертировать в Excel или вручную форматировать. CSV подается как есть.

2. **Критерии из отдельного файла.** Claude использует оба файла: данные из CSV и требования из MD. Это частый рабочий сценарий, когда данные в одном месте, а правила -- в другом.

3. **Обоснование по каждому кандидату.** Claude не просто ранжирует, а объясняет: какой опыт совпал, что не хватает. Это не "Иванова -- 8 баллов", а конкретные причины.

4. **Claude может ошибаться в оценках.** Проверьте пару кандидатов вручную. Если что-то не так -- отличный повод обсудить, что AI-скрининг нужно верифицировать.

5. **50 кандидатов -- это малый объем.** В реальности бывает 200-500. Принцип тот же, просто файл больше.

6. **"Грязные" данные в конце CSV.** Обратите внимание студентов, что последние строки содержат опечатки и нестандартное форматирование. Claude должен справиться и с ними -- как это было бы с реальной выгрузкой из HeadHunter или Google Forms.

## Уточняющие промпты

**Более детальный анализ shortlist:**
```
По каждому из 8 кандидатов в shortlist -- составь таблицу соответствия: слева обязательные требования из вакансии, справа -- что есть у кандидата. Отметь галочками совпадения и крестиками пробелы.
```

**Вопросы для собеседования:**
```
Для топ-3 кандидатов подготовь по 5 вопросов для собеседования. Вопросы должны проверить именно те зоны, где есть сомнения по резюме.
```

**Сравнительная таблица:**
```
Сделай сводную таблицу shortlist: имя, текущая позиция, опыт (лет), размер команды, бюджет, сильные стороны (кратко), слабые стороны (кратко), зарплата, оценка.
```

## Частые проблемы

**Claude выдает больше или меньше 8 кандидатов**
- Укажите точнее: "Ровно 8 человек, не больше"
- Или: "Если сильных кандидатов меньше 8, добавь запасных с пометкой"

**Claude не использует критерии из файла**
- Напомните: "Оценивай строго по критериям из job-requirements.md, используй веса"

**Результат слишком длинный**
- Попросите краткую версию: "Сначала таблица на одну страницу, потом детали только по топ-3"

**CSV не парсится корректно**
- Проверьте кодировку (UTF-8). Если Claude неправильно разбил колонки, укажите: "Разделитель -- запятая, текст в кавычках"

**Claude спотыкается на "грязных" строках**
- Попросите: "Если в строке опечатки или нестандартное форматирование -- все равно проанализируй, исходя из того, что можно понять"
