# Демо: КП за 5 минут

## Время: ~12 минут

## Что здесь

- `offer-template.md` -- шаблон КП компании "Вектор" (консалтинг) с плейсхолдерами
- `call-notes.md` -- заметки со звонка с "ТехноПарк" (Дмитрий Ларионов, коммерческий директор)

## Подготовка

Для этого демо нужен проектный CLAUDE.md из предыдущего демо (`claude-md-intro/project-example.md`). Скопируйте его в текущую папку как `CLAUDE.md`, или убедитесь что он лежит уровнем выше.

Без CLAUDE.md тоже сработает, но Claude не будет знать реквизиты, тарифы и стиль текстов. Можно показать оба варианта для контраста.

## Ход демо

### Шаг 1: Показываем исходные файлы (2 мин)

Открываем `call-notes.md`. Прокручиваем вместе с аудиторией. Обращаем внимание:
- Заметки живые: сокращения, неполные предложения, личные пометки
- Есть конкретика: цифры (оборот 1.2 млрд, EBITDA -4%), имена, проблемы
- Есть контекст: откуда пришел клиент, с кем уже говорил, что для него важно

Затем показываем `offer-template.md`:
- Стандартная структура КП с плейсхолдерами [В СКОБКАХ]
- Обращаем внимание на раздел "Понимание задачи" -- именно там нужна персонализация

### Шаг 2: Генерируем КП (5 мин)

Запускаем Claude Code в папке с файлами. Вводим:

```
Прочитай шаблон КП (offer-template.md) и заметки со звонка (call-notes.md). Сгенерируй персонализированное коммерческое предложение для ТехноПарка. Подбери подходящие услуги из нашего прайса, распиши этапы, рассчитай стоимость. Результат сохрани в файл kp-technopark.md.
```

Пока Claude работает, обращаем внимание аудитории:
- Claude читает оба файла
- Если есть CLAUDE.md -- подтягивает тарифы оттуда
- Подбирает услуги под задачу клиента (скорее всего: операционный аудит + стратегический консалтинг)

### Шаг 3: Разбираем результат (3 мин)

Открываем сгенерированный файл. Проверяем вместе с аудиторией:

- "Понимание задачи" -- отражает ли то что клиент говорил на звонке? Упомянуты ли конкретные проблемы (рентабельность, закупки, ценообразование)?
- Услуги -- подобраны ли правильные? Стоимость соответствует тарифам?
- Этапы и сроки -- разумные ли? Клиент хотел стартовать в марте, результаты к маю
- "Почему мы" -- использована ли рекомендация от "Промтеха"?
- Реквизиты -- правильные?

Если что-то не так -- показываем как уточнить:

```
В разделе "Понимание задачи" добавь про проблему с дублированием функций между двумя заводами. Клиент особенно на это жаловался.
```

### Шаг 4 (бонус, если есть время): PDF (2 мин)

```
Сконвертируй kp-technopark.md в PDF. Добавь шапку с названием компании "Вектор" и датой.
```

Показываем, что Claude может создать PDF прямо из терминала.

## Пример результата

Раздел "Понимание задачи" в готовом КП (примерно):

> ТехноПарк -- производитель упаковки, оборот ~1.2 млрд руб, два завода (Подольск и Тула), ~200 сотрудников. При росте выручки на 18% за прошлый год EBITDA снизилась на 4%. Основные причины: нет прозрачного учёта себестоимости по продуктовым линейкам, ценообразование без единой политики, дублирование закупочных функций между заводами.

Раздел "Стоимость" (примерно):

> | Позиция | Сумма |
> |---------|-------|
> | Операционный аудит (4-6 недель) | 180 000 руб. |
> | Стратегический консалтинг (3 мес) | 750 000 руб. |
> | Итого | 930 000 руб. |

## На что обратить внимание аудитории

1. Claude проанализировал ситуацию клиента и подобрал решение, а не просто заполнил плейсхолдеры
2. Информация из "сырых" заметок превратилась в структурированный документ
3. Весь процесс занял ~5 минут. Вручную такое КП -- 30-60 минут
4. Результат -- черновик. Доработать детали быстрее, чем писать с нуля

## Типичные вопросы

**В: А если заметки на голосовом -- Claude может с ними работать?**
О: Claude Code работает с текстом. Но можно сначала расшифровать голосовое (через Whisper или другой инструмент), а потом отдать Claude. На занятии 3 покажем как подключать внешние сервисы.

**В: Можно ли автоматизировать это полностью?**
О: Да, через Skills (покажем дальше на этом же занятии) или slash-команды (занятие 4). Можно сделать так: кладешь заметки в папку, запускаешь одну команду -- получаешь готовое КП.

**В: А если у нас другой формат КП?**
О: Замените шаблон на свой. Claude работает с любым форматом -- описать структуру достаточно.

## Если что-то пошло не так

| Ситуация | Что делать |
|----------|------------|
| Claude не использует информацию из заметок | Убедитесь что он прочитал оба файла. Можно сказать: "Перечитай call-notes.md и используй оттуда конкретные цифры и проблемы клиента" |
| Стоимость неправильная | Если нет CLAUDE.md с тарифами, Claude может выдумать цены. Укажите прямо: "Операционный аудит стоит 180 000 руб" |
| Результат слишком общий | Попросите: "Сделай раздел 'Понимание задачи' более конкретным. Упомяни оборот 1.2 млрд, падение EBITDA на 4%, два завода в Подольске и Туле" |
| Claude выдумал реквизиты клиента | Нормальное поведение -- в заметках реквизитов нет. Предупредите аудиторию: реквизиты клиента всегда нужно подставлять из реальных документов |
