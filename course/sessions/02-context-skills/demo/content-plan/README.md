# Демо: контент-план из стратегии

## Время: ~15 минут

## Что здесь

Файл `quarterly-strategy.md` -- стратегия Q2 2026 для CloudMetrics (B2B SaaS, мониторинг облачной инфраструктуры). Две страницы: цели, сегменты, каналы, бюджет, рубрики для Telegram-канала.

## Подготовка

В стратегии уже описаны рубрики для Telegram-канала (6 штук), тон коммуникации, целевая аудитория. Claude должен это учесть. Перед демо стоит пролистать стратегию и отметить для себя ключевые блоки.

## Ход демо

### Шаг 1: Показываем стратегию (3 мин)

Открываем `quarterly-strategy.md`. Обращаем внимание аудитории на разделы:
- Цели на Q2 -- из них вытекает контент (например, запуск Enterprise-тарифа = нужен контент про Enterprise)
- Целевые сегменты -- три разных аудитории, контент должен работать на все
- Рубрики Telegram-канала -- уже придуманы, нужно наполнить
- Тон -- "как умный коллега, а не пресс-служба"

### Шаг 2: Генерируем контент-план (5 мин)

```
Прочитай quarterly-strategy.md. Разбей стратегию на 30 публикаций для Telegram-канала на апрель 2026. Используй рубрики из стратегии. Для каждого поста укажи: номер, дату (рабочие дни, 1-2 поста в день кроме воскресенья), рубрику, заголовок, 3-4 тезиса (о чем пост), призыв к действию (CTA). Сохрани в content-plan-april.md.
```

Пока Claude работает, объясняем:
- Claude привязывает контент к стратегии, а не генерирует случайные темы
- Запуск Enterprise-тарифа в апреле -- значит, часть постов будет про это
- Разные рубрики чередуются, чтобы канал не был однообразным

### Шаг 3: Разбираем результат (4 мин)

Открываем контент-план. Проверяем вместе с аудиторией:

- Рубрики чередуются или идут подряд?
- Есть ли привязка к событиям из стратегии? (HighLoad++ в апреле, запуск Enterprise)
- Тезисы конкретные или общие? ("Как компания X сэкономила 40% на облаке" vs "Облачные расходы можно оптимизировать")
- CTA разнообразные или везде одинаковые?
- Тон соответствует описанию в стратегии?

Если что-то не устраивает:

```
Посты 12-15 слишком похожи, все про экономию. Замени пост 13 на что-то из рубрики "Опрос/дискуссия", а пост 14 -- на продуктовое обновление.
```

### Шаг 4 (бонус): Разворачиваем один пост (3 мин)

```
Напиши полный текст для поста #5 из контент-плана. Формат: Telegram-пост до 1500 символов. Стиль -- экспертный, неформальный, с конкретными цифрами. Без эмодзи.
```

Показываем, что из тезисов можно быстро получить готовый текст. А потом этот процесс можно поставить на поток через Skill (покажем в следующем демо).

## Пример результата

Бесплатный аудит -- ссылка в шапке канала |

## На что обратить внимание аудитории

1. Claude берет темы из стратегии, а не из воздуха. Поэтому контент-план привязан к бизнес-целям
2. 30 постов за 5 минут -- это черновик. Редактура нужна, но основная работа сделана
3. Стратегию можно обновить и перегенерировать план. Если цели поменялись, контент адаптируется
4. Это работает для любого канала: блог, email-рассылка, YouTube (заменить формат в промпте)

## Типичные вопросы

**В: А Claude знает нашу отрасль?**
О: Claude знает про большинство отраслей на базовом уровне. Но если нужна глубина -- опишите специфику в CLAUDE.md или в самом промпте.

**В: Контент будет уникальным?**
О: Claude генерирует текст, а не копирует. Но проверяйте факты и цифры -- Claude может выдумать статистику, если ее нет в исходных данных.

**В: Как быть с конкурентами -- Claude не упомянет их?**
О: По умолчанию может упомянуть. Если не хотите -- укажите это в промпте или в CLAUDE.md: "Не упоминать конкурентов по имени".

## Если что-то пошло не так

| Ситуация | Что делать |
|----------|------------|
| Claude сгенерировал меньше 30 постов | Попросите: "Допиши до 30, не хватает постов из рубрик 'Облачные новости' и 'Опрос'" |
| Посты не привязаны к стратегии | Укажите прямо: "В плане должны быть 3-4 поста про запуск Enterprise-тарифа и 2 поста про выход на рынок Казахстана" |
| Все CTA одинаковые | Попросите разнообразить: "CTA должны быть разными: вопрос к аудитории, ссылка на trial, приглашение на вебинар, просьба поделиться опытом" |
| Тон слишком официальный | Напомните: "Пиши как умный коллега в Telegram, а не как корпоративный блог. Можно обращаться на 'ты'" |
