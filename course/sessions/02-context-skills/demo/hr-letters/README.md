# Демо: Skill "HR-письма"

## Время: ~8 минут

## Что здесь

- `SKILL.md` -- навык для генерации HR-писем (повышение, изменение условий, приглашение на работу)
- `sample-data.md` -- данные по 3 сотрудникам с контекстом для писем

## Подготовка

Демо идёт после weekly-report-skill, участники уже понимают что такое Skill и как он устроен. Здесь показываем, что навыки применимы не только к отчётам -- любая повторяющаяся задача с шаблоном подходит.

Если есть проектный CLAUDE.md с реквизитами "Вектора" -- оставьте, он не помешает. Но для HR-писем контекст компании менее критичен.

## Ход демо

### Шаг 1: Показываем SKILL.md и данные (2 мин)

Открываем `SKILL.md`. Обращаем внимание:
- Три типа писем: повышение, изменение условий, приглашение
- У каждого типа свои обязательные поля и свой тон
- Есть правила: формальный стиль, ссылка на ТК РФ, подпись руководителя

Затем показываем `sample-data.md`:
- Три сотрудника с разными ситуациями
- Для каждого указан контекст: должность, стаж, причина письма, конкретные цифры

### Шаг 2: Генерируем письмо о повышении (3 мин)

```
Используй навык hr-letters из SKILL.md. Сгенерируй письмо о повышении для Ильиной Анны из sample-data.md. Сохрани в письмо-повышение-ильина.md.
```

Открываем результат, проверяем:
- Упомянута ли причина повышения (руководство проектом, рост выручки)?
- Указаны ли новая должность и новый оклад?
- Формальный ли тон? Есть ли ссылка на приказ?
- Подпись руководителя на месте?

### Шаг 3: Генерируем письмо другого типа (3 мин)

```
Теперь сгенерируй письмо об изменении условий для Козлова Дмитрия. Сохрани в письмо-условия-козлов.md.
```

Сравниваем два письма:
- Структура другая: здесь про дополнительное соглашение, а не про приказ о повышении
- Тон тот же, но содержание разное
- Skill автоматически подставил правильный шаблон по типу письма

## Пример результата

Письмо о повышении (примерно):

> ООО "Вектор"
>
> Уважаемая Анна Сергеевна!
>
> Руководство ООО "Вектор" благодарит Вас за работу в должности старшего консультанта в течение 2 лет и 4 месяцев. За время работы Вы руководили проектом для АО "ПромЭнерго", который привёл к росту выручки клиента на 12%.
>
> Сообщаем о Вашем переводе на должность ведущего консультанта с 1 марта 2026 г. Новый оклад: 210 000 руб/мес (ранее 165 000 руб/мес).
>
> Основание: приказ № ___ от ___ 2026 г.
>
> Генеральный директор _____________ Кузнецов А.В.

## На что обратить внимание аудитории

1. Один Skill обрабатывает разные типы писем -- не нужно три отдельных навыка
2. Данные сотрудника берутся из файла, а не из головы. В реальной работе это может быть выгрузка из 1С или HR-системы
3. Письма -- заготовки. HR-специалист проверяет содержание и добавляет номер приказа перед отправкой

## Типичные вопросы

**В: А если у нас свой шаблон писем?**
О: Замените инструкции в SKILL.md на свой формат. Или положите шаблон рядом и укажите в инструкциях: "Используй шаблон из template.docx".

**В: Можно ли массово генерировать?**
О: Да. "Сгенерируй письма для всех сотрудников из sample-data.md, каждое в отдельный файл." На занятии 4 покажем, как субагенты делают это параллельно.

## Если что-то пошло не так

| Ситуация | Что делать |
|----------|------------|
| Claude не различает типы писем | Укажите тип явно: "Тип письма -- повышение. Используй шаблон для повышений из SKILL.md" |
| Тон слишком неформальный | Добавьте в промпт: "Официально-деловой стиль, обращение на 'Вы'" |
| Claude выдумывает данные сотрудника | Укажите: "Используй только факты из sample-data.md. Где данных нет, оставь пропуск: ___" |
| Письмо на английском | Укажите в SKILL.md: "Все письма на русском языке" |
