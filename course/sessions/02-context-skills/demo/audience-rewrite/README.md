# Демо: рерайт под аудиторию

## Время: ~13 минут

## Что здесь

Файл `product-description.md` -- техническое описание CDP-платформы DataPulse (~1 страница): архитектура, интеграции, ценообразование. Текст написан для инженеров. Задача -- переписать его для трёх разных аудиторий.

## Подготовка

Файл с описанием намеренно технический: ClickHouse, Kafka, микросервисы, API. Аудитории курса будет понятно, что маркетолог или CEO такой текст читать не станет.

## Ход демо

### Шаг 1: Показываем исходный текст (2 мин)

Открываем `product-description.md`. Прокручиваем, обращаем внимание:
- Текст технический: "микросервисная архитектура (Kubernetes)", "Apache Kafka", "50 000 событий в секунду"
- Полезная информация для всех есть: цены, интеграции, SLA
- Для нетехнической аудитории текст непонятен

Спрашиваем аудиторию: "Вы бы прочитали это до конца? А ваш CEO?"

### Шаг 2: Версия для CEO (3 мин)

```
Прочитай product-description.md. Перепиши описание продукта DataPulse для CEO / собственника бизнеса. Ему важно: сколько стоит, какой ROI, как быстро внедрить, какие риски. Технические детали не нужны вообще. Объем -- не больше 1 страницы. Сохрани в ceo-version.md.
```

Пока Claude работает, объясняем: CEO принимает решение на основе денег и рисков. Ему не нужен ClickHouse. Ему нужно понять: "Плачу X, получаю Y, окупается за Z месяцев".

Открываем результат, проверяем:
- Есть ли ROI или хотя бы намек на него?
- Убраны ли технические термины?
- Есть ли конкретные цифры по стоимости?
- Читается ли за 2 минуты?

### Шаг 3: Версия для маркетолога (3 мин)

```
Теперь перепиши то же описание для директора по маркетингу. Ему важно: какие задачи решает, с какими инструментами интегрируется (CRM, рекламные кабинеты, аналитика), насколько удобно работать, нужен ли разработчик для настройки. Технические детали -- минимум, только если влияют на удобство. Сохрани в marketer-version.md.
```

Открываем результат:
- Упомянуты ли интеграции с Bitrix24, amoCRM, Яндекс.Директ?
- Говорится ли про сегментацию и конструктор сегментов?
- Акцент на удобстве (drag-and-drop, конструктор), а не на скорости обработки?
- Убран ли жаргон вроде "градиентный бустинг" и "Kubernetes"?

### Шаг 4: Версия для инженера (3 мин)

```
Последняя версия -- для технического директора (CTO) или ведущего разработчика. Ему важно: архитектура, стек технологий, API, производительность, безопасность, SLA. Бизнес-метрики можно оставить, но акцент на техническую часть. Сохрани в engineer-version.md.
```

Открываем результат:
- Есть ли детали про стек (ClickHouse, Kafka, Kubernetes)?
- Описаны ли API (REST, GraphQL, SDK)?
- Указана ли производительность (50K событий/сек, медианный ответ 120мс)?
- Упомянута ли безопасность (152-ФЗ, шифрование, RBAC)?

### Шаг 5: Сравнение (2 мин)

Показываем три файла рядом (можно открыть в трех вкладках или попросить Claude):

```
Покажи первый абзац каждой из трех версий (ceo-version.md, marketer-version.md, engineer-version.md) рядом, чтобы было видно разницу в стиле.
```

Обращаем внимание:
- Один продукт -- три разных текста
- Каждый говорит на языке своей аудитории
- Типичная задача маркетинга: адаптация одного сообщения под разные каналы

## Пример результата

Первый абзац версии для CEO (примерно):

> DataPulse объединяет данные о ваших клиентах из CRM, рекламы, аналитики и колл-трекинга. Внедрение от 300 000 руб, срок -- 2-3 недели для базового варианта. Подписка от 120 000 руб/мес при базе до 100 000 клиентов. Хранение данных в РФ, соответствие 152-ФЗ.

Первый абзац версии для маркетолога (примерно):

> DataPulse подключается к Bitrix24, amoCRM, Яндекс.Директу, VK Рекламе и еще 40+ сервисам. Данные о клиентах собираются в единые профили автоматически. Сегменты для кампаний строятся в визуальном конструкторе, без участия разработчиков.

## На что обратить внимание аудитории

1. Claude не просто переписывает -- он выбирает, какую информацию оставить, а какую убрать
2. Для CEO убирает технологии и добавляет ROI. Для инженера наоборот
3. Это работает для любого контента: описание продукта, пресс-релиз, статья, письмо
4. Качество зависит от того, насколько точно вы описали аудиторию в промпте

Перепиши для HR-директора, который выбирает инструмент для внутренней аналитики сотрудников. Ему важно: простота, соответствие 152-ФЗ, сколько стоит на 200 сотрудников.

## Типичные вопросы

**В: А если Claude плохо знает нашу аудиторию?**
О: Опишите аудиторию подробнее в промпте. Или добавьте описание типичных персон в CLAUDE.md -- тогда можно будет просто сказать "перепиши для нашего сегмента Enterprise".

**В: Можно ли сделать это одной командой?**
О: Да, можно попросить сразу три версии. Но лучше по одной -- проще контролировать качество и вносить правки.

## Если что-то пошло не так

| Ситуация | Что делать |
|----------|------------|
| Все три версии похожи друг на друга | Claude не уловил разницу в аудиториях. Уточните: "Версия для CEO должна начинаться с финансовых показателей и не содержать технических терминов. Текущая версия слишком техническая" |
| Слишком длинный текст | Добавьте ограничение: "Не больше 400 слов" или "Уместить на одну страницу A4" |
| Claude добавляет информацию, которой нет в исходнике | Скажите: "Используй только факты из product-description.md. Ничего не додумывай" |
| Claude выдает ответ на английском | Укажите в промпте: "Текст на русском языке" |
