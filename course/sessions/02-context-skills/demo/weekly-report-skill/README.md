# Демо: Skill "Еженедельный отчет"

## Время: ~18 минут

## Что здесь

- `SKILL.md` -- описание навыка в формате Claude Code (имя, описание, инструкции)
- `sales.csv` -- данные по продажам за 2 недели (CSV с разделителем ";")
- `expenses.csv` -- расходы за неделю (CSV с разделителем ";")
- `tasks.md` -- список задач с ответственными и статусами
- `kpi.md` -- план/факт по KPI за февраль

## Подготовка

Перед демо полезно открыть каждый файл данных и бегло показать, что в нем. Аудитория должна видеть: это 4 разных источника, и отчет собирает их в одно место.

Объясните, что в реальной жизни эти файлы обновляются каждую неделю (выгрузки из CRM, бухгалтерии, трекера задач). Skill нужен, чтобы не объяснять Claude каждый раз, что с ними делать.

## Ход демо

### Часть 1: Зачем нужны Skills (3 мин)

Объясняем проблему:
- Каждую пятницу нужно собрать отчет из 4 источников
- Можно каждый раз писать длинный промпт: "прочитай sales.csv, посчитай то-то, потом прочитай expenses.csv, сгруппируй по категориям..."
- Или один раз описать всю процедуру в Skill и потом запускать одной командой

Аналогия: Skill -- инструкция для нового сотрудника. Написал один раз, дальше он делает сам.

### Часть 2: Разбираем SKILL.md (3 мин)

Открываем `SKILL.md`. Проходим по структуре:

```
---
name: weekly-report
description: Собирает еженедельный отчет из данных...
---
```

Обращаем внимание:
- `name` -- по этому имени вызываем навык
- `description` -- Claude видит это описание и понимает когда навык уместен
- Дальше идут инструкции -- обычный текст, описывающий что делать

Проходим по инструкциям:
- Источники данных -- какие файлы читать
- Разделы отчета -- что включать и в каком порядке
- Правила -- "если цифры плохие, так и напиши", "суммы с разделителями тысяч"
- Формат вывода -- имя файла, формат таблиц

### Часть 3: Показываем данные (2 мин)

Быстро показываем каждый файл:

- `sales.csv` -- продажи за 2 недели, есть прошлая неделя для сравнения. Статусы: оплачено, счет выставлен, переговоры, отказ
- `expenses.csv` -- расходы по категориям. Аренда, зарплата, реклама, ПО, прочее
- `tasks.md` -- задачи с галочками. Выполненные, в работе, просроченные
- `kpi.md` -- таблицы план/факт с комментариями

### Часть 4: Запускаем Skill (5 мин)

Для демонстрации есть два пути.

**Вариант А: Через промпт с упоминанием навыка**

```
Используй навык weekly-report из SKILL.md, чтобы собрать еженедельный отчет из файлов в этой папке.
```

**Вариант Б: Установка как полноценный Skill**

Объясняем, что в реальной работе Skill устанавливается в папку `.claude/skills/` и потом доступен автоматически. Для демо можно показать структуру:

```
.claude/
  skills/
    weekly-report/
      SKILL.md
```

После установки Claude подхватывает навык автоматически, когда задача подходит под описание.

### Часть 5: Разбираем результат (3 мин)

Открываем сгенерированный отчет. Проверяем:

- Продажи: правильно ли посчитана сумма оплаченных сделок? (должно быть 423 000 руб за текущую неделю)
- Сравнение с прошлой неделей: прошлая неделя -- 777 000 руб, снижение
- Расходы: общая сумма и разбивка по категориям в таблице
- Задачи: 4 выполнены, 3 в работе, 2 просрочены. Просроченные перечислены с ответственными
- KPI: таблица план/факт. Стоимость лида выше плана, подписчики в Telegram растут медленно
- Итого: есть ли честная оценка? Не приукрашен ли результат?

### Часть 6: Доработка навыка (2 мин)

Показываем, что Skill можно дорабатывать:

```
Добавь в SKILL.md правило: в конце отчета формируй раздел "Три вопроса для планерки" -- конкретные вопросы, которые стоит обсудить с командой на основе данных.
```

Или:

```
Измени SKILL.md так, чтобы отчет также сохранялся в HTML-формате с таблицами.
```

## На что обратить внимание аудитории

1. Skill -- обычный Markdown-файл. Никакого программирования. Пишете инструкции так же, как написали бы для человека
2. Один раз написали -- используете каждую неделю. Данные меняются, формат отчета остается
3. Skill можно делиться с коллегами -- просто отправить файл или положить в общий репозиторий
4. Claude может помочь написать Skill: опишите задачу, попросите оформить как SKILL.md

## Типичные вопросы

**В: Чем Skill отличается от CLAUDE.md?**
О: CLAUDE.md -- контекст (кто вы, как работаете). Skill -- конкретная процедура (что делать и в каком порядке). CLAUDE.md загружается всегда, Skill -- только когда нужен.

**В: Можно ли использовать Skill для других форматов -- Word, Excel?**
О: Да. В инструкциях Skill опишите нужный формат вывода. Claude умеет создавать файлы разных форматов.

**В: Где хранить файлы данных?**
О: Skill описывает из каких файлов читать. Файлы могут лежать в любом месте -- укажите путь в инструкциях.

**В: А если структура CSV изменится?**
О: Обновите Skill. Или напишите инструкцию гибко: "Прочитай CSV с продажами, найди колонки с датой, суммой и статусом (названия могут отличаться)".

## Если что-то пошло не так

| Ситуация | Что делать |
|----------|------------|
| Claude не находит файлы | Убедитесь, что Claude Code запущен в той же папке где лежат файлы. Или укажите полные пути |
| Числа посчитаны неправильно | Claude иногда ошибается в арифметике. Попросите: "Проверь сумму оплаченных сделок. Перечисли все строки со статусом 'оплачено' и их суммы" |
| CSV парсится некорректно | Возможно, разделитель. Укажите: "Файл sales.csv использует разделитель точка с запятой (;), не запятую" |
| Отчет слишком короткий | Skill описывает 6 разделов. Если Claude пропустил какие-то, напомните: "В отчете нет раздела по KPI. Добавь его по инструкции из SKILL.md" |
