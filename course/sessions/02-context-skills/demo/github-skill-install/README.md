# Демо: установка Skill из GitHub

## Время: ~5 минут

## Что показываем

Как найти готовый Skill в открытом репозитории на GitHub и установить его в Claude Code одной командой. Не нужно писать навык с нуля -- можно взять чужой и адаптировать.

## Подготовка

Нужен доступ в интернет (GitHub). Claude Code должен быть запущен.

Демо идёт в конце блока про Skills. Участники уже видели, как создавать навыки вручную (weekly-report, hr-letters). Здесь показываем, что есть готовые.

## Ход демо

### Шаг 1: Объясняем идею (1 мин)

Skills -- это Markdown-файлы. Их легко публиковать и распространять. Люди выкладывают готовые навыки в открытые GitHub-репозитории. Можно установить одной командой.

Формат URL репозитория со Skill:

```
https://github.com/{автор}/{репозиторий}
```

Внутри репозитория лежит SKILL.md (или папка с несколькими навыками).

### Шаг 2: Установка через Claude Code (2 мин)

Просим Claude Code установить Skill:

```
Установи skill из репозитория https://github.com/anthropics/claude-code-skills -- выбери навык для создания документов.
```

Claude Code:
1. Склонирует репозиторий (или скачает нужный файл)
2. Скопирует SKILL.md в `.claude/skills/`
3. Подтвердит установку

Если конкретного репозитория нет или он недоступен, можно показать процесс вручную:

```
Создай папку .claude/skills/doc-generator/ и положи туда SKILL.md со следующими инструкциями: [описание навыка]
```

### Шаг 3: Проверяем что навык работает (2 мин)

```
Какие навыки у тебя сейчас установлены?
```

Claude Code перечислит доступные Skills из `.claude/skills/`. Новый должен быть в списке.

Пробуем запустить:

```
Используй навык [имя навыка] для [задача].
```

## Пример результата

После установки:

```
$ ls .claude/skills/
weekly-report/
hr-letters/
doc-generator/
```

Claude отвечает на запрос "Какие навыки установлены?":

> У меня есть 3 навыка:
> - weekly-report -- собирает еженедельный отчёт из данных по продажам, расходам, задачам и KPI
> - hr-letters -- генерирует HR-письма (повышение, изменение условий, приглашение)
> - doc-generator -- создаёт документы по шаблону (договор, счёт, акт)

## На что обратить внимание аудитории

1. Skill -- просто файл. Установка = скопировать в папку. Не нужен пакетный менеджер
2. Перед использованием чужого Skill стоит прочитать его SKILL.md -- понять что он делает и какие файлы ожидает
3. Чужой навык можно доработать: открыть SKILL.md, изменить инструкции под себя
4. Со временем появится больше репозиториев с готовыми навыками. Следите за каталогом в документации Claude Code

## Если что-то пошло не так

| Ситуация | Что делать |
|----------|------------|
| Claude не может скачать репозиторий | Проверьте доступ в интернет. Или скачайте SKILL.md вручную через браузер и положите в `.claude/skills/` |
| После установки навык не появляется | Проверьте структуру: файл должен лежать в `.claude/skills/{имя}/SKILL.md`. Убедитесь что файл называется именно SKILL.md |
| Навык работает не так, как ожидалось | Откройте SKILL.md и прочитайте инструкции. Возможно, навык ожидает файлы определённого формата или CLAUDE.md с настройками |
| Нужного навыка нет в репозитории | Попросите Claude создать: "Напиши SKILL.md для [описание задачи] и установи в .claude/skills/" |
