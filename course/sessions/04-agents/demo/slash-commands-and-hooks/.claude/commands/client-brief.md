Подготовь бриф по клиенту или сотруднику перед встречей.

Аргумент: $ARGUMENTS (имя сотрудника или название компании)

Действия:
1. Проверь файл ../360-review-reports/evaluation-data.csv — найди все оценки, связанные с этим человеком (как оцениваемого и как оценщика)
2. Проверь файл ../meeting-protocol/meeting-transcript.md — есть ли упоминания этого человека в расшифровке совещания
3. Если данных по конкретному имени нет, выведи список доступных сотрудников из evaluation-data.csv

Формат результата:
- Имя, должность
- Сводка по 360-оценке: средние баллы по категориям, сильные стороны, зоны роста
- Как оценивают коллеги vs руководители vs подчинённые — есть ли расхождения
- Упоминания в расшифровке совещания: какие задачи обсуждались, какие решения приняты
- Рекомендации: на что обратить внимание при встрече

Сохрани в файл briefs/$ARGUMENTS-[дата].md
