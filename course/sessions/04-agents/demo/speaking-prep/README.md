# Демо: Подготовка к выступлению

## Что делаем

Claude Code получает тему, тезисы и контекст спикера (topic-and-outline.md) и создаёт три документа: HTML-презентацию на 15 слайдов, двухстраничный раздаточный материал (handout) и спикерские заметки с таймингом. Субагенты работают параллельно, каждый над своим документом.

## Время выполнения

- Claude (Opus 4.6): 3-5 минут
- GLM 4.7: 5-8 минут (три параллельных субагента, но задача нетяжёлая для каждого)

## Подготовка

Файл `topic-and-outline.md` содержит:
- Контекст спикера: Алексей Морозов, COO логистической компании, 12 лет в операционке
- Формат: 20 минут доклад + 10 минут вопросы, конференция "Операционный директор 2026"
- 7 тезисов -- конкретные кейсы внедрения AI: три сработали, два провалились, один заморозили, плюс общий вывод

Тезисы написаны с цифрами: 14% снижение простоев, 2.8 млн стоимость разработки, 73% точность компьютерного зрения. Claude будет вытаскивать эти цифры и расставлять по слайдам.

## Что вводим в Claude Code

```
Прочитай topic-and-outline.md — это тезисы для выступления на конференции. Подготовь три документа:

1. Презентация (15 слайдов, HTML с Chart.js):
   - Титульный слайд с именем спикера и темой
   - По 1-2 слайда на каждый из 7 тезисов
   - Итоговый слайд с выводами
   - Слайд с контактами
   - Дизайн: светлый фон, минималистичный, крупные цифры
   - Где есть цифры — графики или крупные числа на весь слайд
   - Минимум текста: не больше 3 строк на слайд

2. Раздаточный материал (handout, PDF-ready markdown):
   - 2 страницы максимум
   - Краткое резюме каждого кейса: что сделали, результат, стоимость
   - Таблица: кейс / статус / ROI / срок
   - Контакты спикера

3. Спикерские заметки (speaker-notes.md):
   - Для каждого слайда: что говорить, ключевые фразы
   - Тайминг: сколько минут на каждый блок (всего 20 минут)
   - Где сделать паузу, где задать вопрос залу
   - Возможные вопросы из зала и короткие ответы

Используй субагентов для параллельной работы над тремя документами.
```

## Что происходит

1. Claude читает тезисы и контекст спикера
2. Запускает три субагента параллельно: один делает презентацию, второй -- handout, третий -- спикерские заметки
3. Каждый субагент получает одни и те же тезисы, но задачи у них разные
4. Результат: три файла (presentation.html, handout.md, speaker-notes.md)

Обратите внимание: субагенты не зависят друг от друга, поэтому работают по-настоящему параллельно (в отличие от демо с презентацией, где второй ждёт первого).

## Пример результата

**Презентация** (presentation.html):
- Слайд 3: крупная цифра "14%" на весь экран, подпись "снижение простоев за 4 месяца"
- Слайд 5: красный фон, "1.5 млн потерь" -- провал с автоматической маршрутизацией
- Слайд 11: график тональности 2000 тикетов по месяцам

**Handout** (handout.md):
```
| Кейс                     | Статус     | ROI              | Срок      |
|--------------------------|------------|------------------|-----------|
| Прогноз загрузки складов | Работает   | Окупился за 5 мес| 4 мес     |
| Маршрутизация            | Провал     | -1.5 млн         | 3 недели  |
| Скоринг поставщиков      | Работает   | 8-11% экономия   | 6 мес     |
| Внутренний чат-бот       | Работает   | -40% нагрузки HR | 3 недели  |
| Компьютерное зрение      | Заморожен  | —                | —         |
| Анализ тикетов           | Работает   | -22% жалоб       | 2 мес     |
```

**Спикерские заметки** (speaker-notes.md):
```
Слайд 1 (0:00-0:30): Титульный
"Добрый день. Меня зовут Алексей Морозов, я операционный директор в [компания].
За последние 2 года мы потратили около 12 миллионов на AI-эксперименты.
Часть денег отбились, часть — нет. Сегодня расскажу, где граница."

Слайд 2-3 (0:30-3:00): Прогноз загрузки складов
Ключевая фраза: "14% — это 40 машин в месяц, которые не стоят пустыми."
[пауза, дать осмыслить цифру]
...
```

## Что может пойти не так

**Презентация перегружена текстом.** Добавьте: "На слайде максимум 3 строки текста. Лучше одна цифра крупно, чем абзац мелким шрифтом."

**Спикерские заметки слишком формальные.** Тезисы написаны живым языком, но Claude может "причесать" их в канцелярит. Попросите: "Спикерские заметки пиши разговорным языком, как будто человек реально стоит на сцене."

**Тайминг не сходится.** Claude иногда выделяет по 3 минуты на каждый из 7 тезисов, и получается 21 минута без вступления и заключения. Проверьте, что общее время не превышает 20 минут.

**Handout получается на 4 страницы вместо 2.** Укажите: "Handout строго на 2 страницы. Используй таблицу, а не описательный текст."
