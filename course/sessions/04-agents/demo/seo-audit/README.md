# Демо: SEO-аудит блога (80 статей)

## Что делаем

Claude Code анализирует CSV с 80 статьями блога: проверяет актуальность, объём, наличие мета-описаний, давность обновления. Запускает субагентов для параллельного аудита. На выходе -- список с приоритетами: какие статьи обновить, какие удалить, о чём написать новые.

## Время выполнения

- Claude (Opus 4.6): 2-4 минуты
- GLM 4.7: 4-6 минут (80 строк CSV обрабатываются без проблем, но анализ каждой статьи требует рассуждений)

## Подготовка

Файл `blog-articles.csv` содержит 80 записей: заголовок, URL, дата публикации, количество слов, ключевое слово, мета-описание, дата обновления.

Данные специально неоднородные (как в реальном блоге):
- Статьи 2022 года, ни разу не обновлённые
- Короткие обзоры на 400-700 слов (тонкий контент)
- ~15 статей без мета-описания
- Хорошие статьи 2024 года на 2500+ слов с мета-описаниями
- Устаревшие темы ("тренды CRM 2023", "обзор Salesforce для российского рынка")
- Свежие статьи про AI, unit-экономику, RevOps

## Что вводим в Claude Code

```
В файле blog-articles.csv — 80 статей из корпоративного блога. Проведи SEO-аудит и рассортируй статьи по приоритетам.

Критерии аудита:
- Актуальность: статьи старше 18 месяцев без обновления — кандидаты на ревизию
- Объём: статьи короче 800 слов — тонкий контент, скорее всего, не ранжируется
- Мета-описания: если нет — нужно добавить
- Тема: устаревшие тренды ("2023"), уход сервисов с рынка (Salesforce в России) — обновить или удалить
- Ключевые слова: проверь, нет ли каннибализации (несколько статей на одно ключевое слово)

Используй субагентов для параллельной обработки — раздели статьи на группы.

Результат — отчёт с тремя списками:
1. ОБНОВИТЬ (приоритет: высокий/средний/низкий) — с указанием, что именно обновить
2. УДАЛИТЬ или объединить — почему
3. НАПИСАТЬ НОВЫЕ — какие темы не покрыты, исходя из ключевых слов и пробелов

Сохрани в файл seo-audit-report.md
```

## Что происходит

1. Claude читает CSV и оценивает общую картину
2. Разбивает 80 статей на группы (по 15-20) и запускает субагентов для параллельной обработки
3. Каждый субагент проверяет свою группу по критериям
4. Основной Claude собирает результаты в один отчёт
5. Результат: файл с приоритизированным списком действий

Claude обнаруживает каннибализацию (несколько статей про CRM для разных ниш), правильно определяет "мёртвые" темы (обзор Salesforce для российского рынка после 2022), а тонкий контент (400-700 слов) отправляет в "обновить" или "удалить".

## Пример результата

Файл `seo-audit-report.md`:

**Обновить** (~25-30 статей):
- Высокий приоритет: "Как выбрать CRM для малого бизнеса в 2024 году" -- обновить заголовок и содержание на 2026, статья хорошая (2800 слов), но заголовок устарел
- Средний приоритет: 15 статей без мета-описания -- добавить
- Низкий приоритет: мелкие правки по датам и ссылкам

**Удалить/объединить** (~10-15 статей):
- "Обзор Мегаплан: возможности и цены" (400 слов), "Обзор RetailCRM" (600 слов), "Обзор Pipedrive" (500 слов) -- тонкий контент, объединить в одну сравнительную статью
- "Обзор Salesforce для российского рынка" -- Salesforce ушёл из России, статья неактуальна
- "Топ-10 трендов CRM в 2023 году" -- устарела

**Написать новые** (5-8 тем):
- AI в CRM (тема горячая, статьи нет)
- Сравнение российских CRM 2026
- No-code автоматизация продаж

## Что может пойти не так

**Claude делает всё в одном потоке.** С 80 статьями это не страшно, работает за 3-4 минуты. Но для демо параллелизма можно явно указать: "Раздели статьи на 4 группы по 20 и обработай каждую группу отдельным субагентом".

**Аудит слишком поверхностный.** Claude не может открыть URL и прочитать содержимое статей: он работает только с CSV. Объясните студентам, что это аудит по метаданным. Для полного аудита нужен MCP с доступом к сайту (тема занятия 3).

**Слишком длинный отчёт.** Если Claude расписывает каждую из 80 статей отдельным абзацем, добавьте: "Формат -- таблица. Для каждой статьи: заголовок, действие, приоритет, что сделать (1 строка)."
