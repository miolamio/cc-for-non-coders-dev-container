# Демо: Протокол встречи из расшифровки

## Что делаем

Claude Code берёт расшифровку 15-минутного совещания (meeting-transcript.md) с таймкодами, вытаскивает из разговорного текста все решения, задачи, дедлайны и ответственных. На выходе -- структурированный протокол и готовое сообщение для Telegram-чата команды.

Два субагента работают параллельно: один формирует формальный протокол, другой пишет краткое сообщение для мессенджера.

## Время выполнения

- Claude (Opus 4.6): 1-2 минуты
- GLM 4.7: 2-3 минуты

## Подготовка

Файл `meeting-transcript.md` содержит расшифровку совещания с таймкодами (формат [00:03:45]) на тему запуска нового направления (внедрение AI). Шесть участников, естественная речь: перебивания, вопросы не по теме, решения разбросаны по тексту. Это сделано специально -- студенты увидят, что Claude справляется с "грязными" данными.

## Что вводим в Claude Code

```
Прочитай расшифровку совещания в meeting-transcript.md и подготовь два документа:

1. Формальный протокол совещания:
   - Дата, участники
   - Повестка
   - Принятые решения (пронумерованные)
   - Задачи: кто, что, до какого числа
   - Открытые вопросы (что не решили)
   - Дата следующего совещания

2. Сообщение для Telegram-чата команды:
   - Короткое, неформальное
   - Только суть: что решили, кому что делать, когда следующий звонок
   - Формат: текст для одного сообщения, не длиннее 15 строк

Оба документа сохрани как отдельные файлы.
```

## Что происходит

1. Claude читает расшифровку и определяет, что задача делится на две независимые части
2. Запускаются два субагента: один пишет формальный протокол, другой -- Telegram-сообщение
3. Оба работают параллельно, в терминале видно обе задачи одновременно
4. Результат: два файла (protocol.md и telegram-message.md)

Обратите внимание: в расшифровке есть отвлечения (Катя спрашивает про рассылку, Артём про Свету из юридического). Claude корректно их отфильтровывает. Решения в тексте не помечены словом "решение", Claude определяет их по контексту. Некоторые дедлайны подразумеваются ("до пятницы", "на следующей неделе"), Claude вычисляет конкретные даты от даты совещания.

## Пример результата

**Протокол** (protocol.md):
- 5-7 пронумерованных решений (утверждение пакетов, бюджет маркетинга 250 тыс., вебинар до 7 марта, прозвон с 3 марта и т.д.)
- 8-10 задач с ответственными и сроками
- 2-3 открытых вопроса (название направления, согласование кейса с Метро Кэш)

**Telegram-сообщение** (telegram-message.md):
```
Коротко по планёрке 14.02:

Утвердили 3 пакета: аудит (800к), внедрение (от 3М), подписка (200к/мес).
Денис готовит описания до пятницы.
Вебинар до 7 марта, Денис спикер, Катя организует.
Бюджет на маркетинг: 250к на первый месяц.
Артём начинает прозвон с 3 марта.
Следующий звонок — 21.02 в 12:00.

Подробный протокол скину файлом.
```

## Что может пойти не так

**Claude пропускает неявные решения.** Например, "Давай в 12:00. Все? -- Ок" -- это решение о времени следующего совещания. Если пропускает, уточните: "Проверь, нет ли неявных решений -- согласований времени, перенесённых встреч".

**Telegram-сообщение получается слишком формальным.** Добавьте: "Telegram-сообщение пиши так, как написал бы руководитель в рабочий чат -- без 'уважаемые коллеги' и 'по итогам совещания'."

**Даты вычислены неправильно.** Claude иногда ошибается с "следующей средой" или "до пятницы". Проверяйте вместе со студентами -- это хороший повод обсудить, что результаты агента нужно проверять.
