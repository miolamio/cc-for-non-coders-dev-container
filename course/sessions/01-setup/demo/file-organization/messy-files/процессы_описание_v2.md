# Как у нас устроены процессы (v2, сентябрь 2023)

## Пресейл
1. Получили лид (сайт, рекомендация, Telegram)
2. Первый звонок: выяснить задачу, бюджет, сроки (30 мин)
3. Если подходит -- готовим КП (2-3 дня)
4. Презентация КП клиенту
5. Переговоры, согласование
6. Договор + предоплата

## Продакшн
1. Kickoff: знакомство с командой, обсуждение деталей
2. Аналитика: 1-2 недели, на выходе -- ТЗ или user stories
3. Дизайн: 2-4 недели, итерации с клиентом
4. Разработка: 4-8 недель, промежуточные демо каждую неделю
5. Тестирование: 1 неделя
6. Запуск

## Постпродакшн
1. Гарантийный период: 1 месяц, правим баги бесплатно
2. Поддержка: по запросу, почасовая оплата (3500 ₽/час)
3. Развитие: если клиент хочет новые фичи -- отдельный договор

## Проблемы текущих процессов
- Нет нормального шаблона ТЗ, каждый раз пишем с нуля
- Клиенты не понимают, когда нужна обратная связь (задерживают согласование)
- Нет базы знаний: если человек уходит, его опыт теряется
- Ретро проводим не всегда

## Что хотим поменять
- Шаблоны для всего: ТЗ, КП, договор, акт
- Автоматические напоминания клиенту (через amoCRM?)
- Wiki в Notion с описанием всех процессов
- Обязательное ретро после каждого проекта
