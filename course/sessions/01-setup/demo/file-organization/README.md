# Демо: Систематизация файлов

## Что здесь

Папка `messy-files/` -- 25 файлов, которые "накопились за 3 года" у руководителя небольшой digital-студии. Заметки со звонков, протоколы встреч, шаблоны, чек-листы, планы, стратегии, списки задач -- всё вперемешку, без структуры.

Файлы написаны от лица одного человека и связаны общей историей (одни и те же клиенты, сотрудники, проекты). Claude может найти эти связи.

Имена файлов тоже хаотичные -- кириллица и латиница, пробелы и подчёркивания, .txt и .md вперемешку. Типичная картина.

## Время

На Claude Pro/Max: ~1-2 минуты (чтение всех файлов и генерация INDEX.md).
На GLM 4.7: ~2-4 минуты.

Claude будет читать каждый файл по очереди -- в терминале будет видно, как он открывает их один за другим. Можно прокомментировать: "он не по названию угадывает, а реально читает содержимое".

## Цель демо

Показать, что Claude Code может прочитать все файлы в папке, понять содержимое, предложить структуру и создать навигацию. Не нужно открывать каждый файл и решать, куда его положить.

## Подготовка

1. Открыть терминал в папке `file-organization/`
2. Запустить Claude Code
3. Можно показать `ls messy-files/` -- студенты увидят хаос в именах файлов

## Промпт для демонстрации

```
В папке messy-files лежат 25 файлов -- заметки, шаблоны, чек-листы, протоколы, всё вперемешку. Накопилось за 3 года.

Прочитай все файлы, разберись в содержимом и:
1. Предложи структуру папок для организации (по типу документа или по теме -- как будет логичнее)
2. Создай файл INDEX.md с оглавлением: для каждого файла -- краткое описание содержимого и рекомендованная папка
3. Не перемещай файлы пока -- только покажи план
```

## Что произойдёт

Claude Code:
1. Прочитает все 25 файлов (по одному, последовательно)
2. Проанализирует содержимое каждого файла
3. Предложит структуру, например:
   - `clients/` -- заметки со звонков и протоколы по клиентам
   - `templates/` -- шаблоны документов (КП, ТЗ, договор, письма)
   - `checklists/` -- чек-листы
   - `reports/` -- отчёты, ретроспективы, обратная связь
   - `strategy/` -- планы, KPI, стратегия, бюджеты
   - `hr/` -- вакансии, онбординг, список подрядчиков
   - `marketing/` -- контент-план, расходы на маркетинг, конкуренты
   - `misc/` -- личные заметки, идеи, книги
4. Создаст INDEX.md с таблицей: файл, описание, рекомендованная папка

## Пример результата

INDEX.md будет выглядеть примерно так (первые 5 записей из 25):

```markdown
# Индекс файлов

## Предложенная структура папок

- `clients/` -- заметки со звонков и протоколы встреч с клиентами
- `templates/` -- шаблоны документов
- `checklists/` -- чек-листы
- `reports/` -- отчёты и ретроспективы
- `strategy/` -- планы, KPI, бюджеты

## Файлы

| Файл | Описание | Папка |
|------|----------|-------|
| заметки со звонка ретейлгрупп.txt | Звонок с РетейлГрупп, ноябрь 2023. Обсуждение редизайна сайта, бюджет 2-3 млн, сроки до Q1 2024 | clients/ |
| чеклист_запуск_проекта.md | Чек-лист для запуска нового проекта: от подписания договора до передачи в поддержку | checklists/ |
| шаблон КП.md | Шаблон коммерческого предложения с плейсхолдерами для названия, дат, сумм | templates/ |
| Отчёт_Q3_2023_драфт.md | Черновик квартального отчёта Q3 2023: выручка 4.82 млн (план 5 млн), 3 проекта, 2 новых клиента | reports/ |
| ВАЖНО прочитать.txt | Стратегические заметки на начало 2023: зависимость от ТехноСтрой (30% выручки), план диверсификации | strategy/ |
| ... | ... | ... |
```

## На что обратить внимание студентов

1. Claude прочитал все 25 файлов. Не по названиям, а по содержимому. Файл "ВАЖНО прочитать.txt" -- это стратегические заметки, а не что-то срочное. Claude это понял.

2. Нашёл связи между файлами. Одни и те же клиенты (РетейлГрупп, ИнноТех, ТехноСтрой) упоминаются в заметках, протоколах, отчётах. Claude может предложить организацию по клиентам.

3. Мы специально попросили "не перемещай пока" -- сначала посмотреть план, потом одобрить. Студенты учатся использовать Plan Mode.

4. Можно продолжить. После одобрения: "Выглядит хорошо, разложи файлы по папкам" -- и Claude создаст структуру и переместит файлы.

## Второй промпт (если останется время)

```
План хороший. Создай папки и разложи файлы. После этого обнови INDEX.md с финальной структурой и добавь ссылки на файлы.
```

Claude создаст папки, переместит файлы и обновит INDEX.md.

## Вариации

Можно показать другой подход к формулировке:

```
У меня бардак в рабочих файлах. Разберись и наведи порядок.
```

Более размытый промпт. Claude всё равно справится, но результат будет менее предсказуемым. Хороший момент для обсуждения "как формулировать задачи" в блоке "Разбор и закрепление".

## Если что-то пошло не так

Claude прочитал не все файлы (остановился на 10-15):
```
Ты прочитал не все файлы. В папке messy-files 25 файлов. Прочитай оставшиеся и дополни INDEX.md.
```

INDEX.md получился слишком общим (описания вроде "заметки" без конкретики):
```
Описания слишком общие. Для каждого файла укажи конкретику: какой клиент, какой период, какие суммы, какие решения. Перепиши INDEX.md.
```

Claude сразу начал перемещать файлы, не спросив:
```
Стоп. Верни файлы на место (в messy-files). Сначала покажи план -- что куда переместишь. Я посмотрю и скажу, если нужно что-то поменять.
```

Claude предложил слишком много папок (10+):
```
Слишком много папок, сложно ориентироваться. Сведи к 5-6 основным категориям. Лучше одна папка "разное", чем 5 папок по одному файлу.
```

## Возможные проблемы

| Ситуация | Что делать |
|----------|------------|
| Claude читает файлы слишком долго | Это нормально для 25 файлов -- он читает по одному. Прокомментировать: "он читает каждый файл, не по имени, а по содержимому" |
| Предложил неудачную структуру | Обучающий момент: "мы не согласны -- давайте уточним". Попросить: "Лучше организуй по типу документа, а не по клиенту" |
| Не создал INDEX.md | Явно попросить: "Создай файл INDEX.md в папке file-organization/" |
| При перемещении потерял файлы | Claude использует `mv`. Если что-то пошло не так -- Ctrl+C, посмотреть что получилось, при необходимости откатить |
